# 更新日志

## 2026-07-09 插件可选化：移除 local_node 硬依赖

### 核心原则
插件安装了加功能，不安装不影响骨架正常运行。所有插件（包括 local_node）都是可选的。

### 后端改动

#### 1. routers.py — local_node_status 端点静默化
- 在尝试 import eta_node 之前，先检查 `_loaded_in_process` 集合
- 如果 local_node 未加载，直接返回 `{"available": false, "message": "local_node 插件未安装或未启用"}`
- 不再尝试查询数据库或 import，避免 ImportError 日志

#### 2. main.py — SPA fallback 路径动态化
- 移除硬编码的 `/local_node/` 路径排除
- 改为从 `_loaded_plugins` 列表动态构建 `plugin_prefixes`
- 任何已加载的插件路径都会被排除在 SPA fallback 之外

#### 3. config.py — 默认不启用任何插件
- `plugins_enabled` 默认值从 `["local_node", "bili_audio"]` 改为 `[]`
- 用户安装后需要主动在插件管理页面启用所需插件

### 前端改动

#### 4. plugins store — syncLocalNode 静默处理
- 当 API 返回 `available: false` 时，静默设置 `localNode = { available: false }`
- 不再显示"请前往插件管理启用"之类的错误提示
- `hasNodePlugin` 改为检查是否有名称含 `node` 的插件已启用且已加载

#### 5. NodesView.vue — 隐藏不可用的本地节点卡片
- 本地节点卡片添加 `v-if="localNode?.available"` 条件
- 移除"正在获取本地节点状态..."加载状态
- 移除"本地节点不可用"错误状态和"前往插件管理"按钮
- 插件未安装时，整个本机节点区域完全不可见

#### 6. plugins/index.js — 动态导入 local_node
- 将 `import local_node from './local_node'` 静态导入改为 `import.meta.glob` 动态导入
- 插件模块缺失时不影响基础程序运行

### 分支
- 开发分支：`dev-260709-plugin-optional`
- 基于 main 分支

## 2026-07-09 在线插件注册表 + 版本兼容性控制

### 新功能：在线插件注册表

#### 1. plugins.json 插件索引
- 项目根目录新增 `plugins.json`，声明所有可用插件的元数据
- 包含：name、package、display_name、description、version、author、eta_web_version、directory、category、icon
- 通过 GitHub Raw URL 获取：`https://raw.githubusercontent.com/kono707da/EtaMusicPlayer/main/plugins.json`

#### 2. 在线注册表客户端 (online.py)
- `OnlinePlugin` 类：解析插件元数据，`is_compatible()` 方法检查骨架版本兼容性
- `OnlineRegistry` 类：从 GitHub 获取 plugins.json，5 分钟缓存，支持强制刷新
- `compare_with_local()` 方法：合并在线信息与本地安装状态，返回 can_install/can_update/update_available

#### 3. 插件安装器 (installer.py)
- `install_plugin()`：下载仓库 zip → 解压插件目录 → 安装 Python 依赖 → 同步注册表
- `update_plugin()`：覆盖安装已安装的插件
- 从 GitHub archive URL 下载整个仓库 zip，按 `directory` 字段提取对应子目录
- 安装后自动将插件目录加入 sys.path 并同步数据库注册表

#### 4. 版本兼容性控制
- `eta_web.__version__` = "1.0.0"：骨架版本号
- 每个插件 `PLUGIN_META` 新增 `eta_web_version` 字段（如 `">=1.0.0,<2.0.0"`）
- `plugins.json` 中每个插件声明 `eta_web_version` 语义化版本约束
- 使用 `packaging.specifiers.SpecifierSet` + `packaging.version.Version` 进行兼容性检查
- 不兼容时禁止安装/更新，前端显示红色不兼容标识

#### 5. 后端 API 新增
- `GET /api/plugins/online` — 获取在线插件列表（合并本地状态和兼容性）
- `GET /api/plugins/online/status` — 检查在线注册表连接状态
- `POST /api/plugins/online/refresh` — 强制刷新在线注册表缓存
- `POST /api/plugins/online/{name}/install` — 从 GitHub 安装插件
- `POST /api/plugins/online/{name}/update` — 从 GitHub 更新插件

#### 6. 前端插件管理页面重构
- 新增「在线插件」标签页，与「已安装」标签页并列
- 在线插件列表展示：插件名、分类、在线版本、本地版本、兼容性、说明、操作
- 兼容性状态：✅ 兼容（绿色）/ ❌ 不兼容（红色）/ ⚠️ 未知（黄色）
- 操作按钮：安装 / 更新 / 已是最新 / 不兼容 / 仅本地
- 连接状态指示器：WiFi 图标 + 骨架版本号
- 安装/更新确认对话框，显示版本和操作说明

#### 7. 配置新增
- `config.yaml` 新增 `plugin_registry_url` 和 `plugin_repo_archive_url`
- `requirements.txt` 新增 `packaging>=23.0` 依赖

## 2026-07-09 Monorepo 项目拆分 + Docker 部署

### 架构重构

#### 1. 项目拆分为四个子项目（Monorepo）
- **web/** — Web 骨架子项目（前端 Vue SPA + 后端 FastAPI，单进程单端口）
  - `web/frontend/` — Vue 3 前端（从 `frontend/` 迁移）
  - `web/backend/eta_web/` — FastAPI 后端骨架（从 `backend/app/` 拆分）
  - `web/backend/eta_web/plugins/` — 插件加载器（sys.path 注入 + 动态导入）
  - `web/backend/eta_web/plugins_manager/` — 插件管理器（注册表、启用/禁用、API）
  - `web/backend/eta_web/plugins/registry.py` — 插件注册表常量（避免循环导入）
- **node/** — 无网页节点程序（local_node 独立部署）
  - `node/eta_node/` — 从 `backend/app/plugins/local_node/` 拆分
  - `node/eta_node/standalone.py` — 独立运行模式（端口 8001）
  - `node/eta_node/config.py` — 独立配置（环境变量前缀 `ETA_NODE_`）
  - `node/eta_node/security.py` — 独立认证模块
- **plugins/** — 插件合集
  - `plugins/asmr_one/eta_asmr/` — 从 `backend/app/plugins/asmr_one/` 拆分
  - `plugins/bili_audio/eta_bili/` — 从 `backend/app/plugins/bili_audio/` 拆分
  - `plugins/shared/eta_shared/` — 共享工具模块（tag_writer）
- **android/** — 安卓访问端（Kotlin + Jetpack Compose 骨架）

#### 2. 包命名规范
- `eta_web` — Web 骨架后端
- `eta_node` — 节点程序
- `eta_asmr` — ASMR 插件
- `eta_bili` — B站音频插件
- `eta_shared` — 共享工具

#### 3. 插件发现机制
- `PLUGIN_PACKAGE_MAP`：插件名 → Python 包名映射
- `PLUGIN_SEARCH_PATHS`：从 `config.yaml` 的 `plugins_dirs` 解析，相对于 `BASE_DIR`
- 启动时将插件目录加入 `sys.path`，通过 `importlib.import_module` 动态加载
- 跨插件依赖使用 `try/except ImportError` 降级处理

#### 4. 节点双模式运行
- **插件模式**：被 Web 骨架加载，挂载在 `/local_node` 路径
- **独立模式**：`python -m uvicorn eta_node.standalone:app --port 8001`，挂载在 `/` 路径

## 2026-07-08 新增bili_audio插件 - B站视频音频提取入库

### 新增功能

#### 1. bili_audio 插件
- 从B站视频提取音频，自动写入标签并入库到本地节点
- 支持BV号、av号、完整B站链接解析

- 视频信息解析（标题、UP主、封面、分P列表）

- 流式下载音频（m4s格式），支持进度回调

- 支持代理和SESSDATA Cookie（用于需要登录的视频）

#### 3. 下载引擎 (downloader.py)
- 完整下载流程：m4s下载 → ffmpeg转码 → 标签写入 → 封面保存 → local_node扫描

- 未选中或只右键一首时，编辑这一首

### 后端新增

#### API: `POST /api/metadata/batch-update`
- 一次请求批量更新多个字段
- 请求体：`{ track_ids: [int], updates: { field: value } }`
- 所有 track_ids 都会被写入 updates 里的全部字段值
- 返回：`{ updated: int, fields: [str], skipped: [{field, reason}] }`
- 跳过项包含失败原因（如非法整数值、文件回写失败等）

#### 函数: `batch_update_multi_fields`
- 在 `metadata_editor.py` 中新增
- 支持一次写入多个字段，类型转换 + 文件回写 + 事务保护
- 跳过非法值并记录原因，不中断其他字段的处理

- 抽屉支持 ESC 关闭（含未保存提示拦截）
