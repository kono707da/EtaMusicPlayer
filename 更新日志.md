## 2026-07-08 新增bili_audio插件 - B站视频音频提取入库

### 新增功能

#### 1. bili_audio 插件
- 从B站视频提取音频，自动写入标签并入库到本地节点
- 支持BV号、av号、完整B站链接解析
- 支持多分P视频选择
- 支持多种音质选择：64kbps / 132kbps / 192kbps（推荐）/ Hi-Res无损
- 支持输出格式：MP3 / M4A(AAC)
- 后台线程下载，支持进度追踪、取消、删除
- 下载完成后自动触发 local_node 扫描入库

#### 2. B站API客户端 (bili_client.py)
- 视频信息解析（标题、UP主、封面、分P列表）
- 音频流URL获取（DASH格式 + durl格式双回退）
- 流式下载音频（m4s格式），支持进度回调
- 封面图片下载
- 支持代理和SESSDATA Cookie（用于需要登录的视频）

#### 3. 下载引擎 (downloader.py)
- 完整下载流程：m4s下载 → ffmpeg转码 → 标签写入 → 封面保存 → local_node扫描
- 文件名非法字符清理（含【】《》（）等全角字符）
- 保存路径结构：{监控目录}/{子目录}/{UP主}/{视频标题}/{分P标题}.{格式}
- 标签写入：标题、艺术家(UP主)、专辑(视频标题)、专辑艺术家、来源URL(bilibili链接)、封面
- ffmpeg自动检测（PATH + 常见安装路径）

#### 4. 前端界面
- 下载页面：链接输入 → 解析 → 选择分P/音质/格式/保存目录 → 下载
- 任务列表页面：实时进度显示（5秒轮询）、取消/删除操作
- 设置页面：代理地址、SESSDATA Cookie配置

#### 5. REST API (/api/bili)
- POST /parse - 解析B站视频链接
- POST /downloads - 创建下载任务
- GET /downloads - 分页查询任务列表
- GET /downloads/{id} - 获取任务详情
- POST /downloads/{id}/cancel - 取消任务
- DELETE /downloads/{id} - 删除任务
- GET /settings - 获取设置
- PUT /settings - 更新设置
- GET /target-nodes - 获取可用监控目录

### Bug 修复

#### 6. B站API fnval参数缺失
- 添加 fnval=16 参数请求DASH格式音频流
- 增加 durl 格式回退支持（未登录时API返回durl而非dash）

#### 7. 前端api.js import路径修复
- 修复 `../../api/request` 不存在的问题，改用 axios 直接创建客户端

### 分支

- 基于 main 分支

## 2026-07-08 插件解耦 + 来源URL写入标签

### 架构优化

#### 1. tag_writer 提取为公共模块
- 将 `app/plugins/asmr_one/tag_writer.py` 提取为 `app/utils/tag_writer.py`
- local_node 的 `metadata.py` 不再依赖 asmr_one 插件，改为从 `app.utils.tag_writer` 导入
- asmr_one 的 `tag_writer.py` 保留为兼容层（re-export），旧导入路径仍可用
- asmr_one 的 `downloader.py` 改为从 `app.utils.tag_writer` 导入
- **解耦效果**：local_node 可独立运行，禁用 asmr_one 插件后批量封面/歌词写入功能不受影响

#### 2. 插件间依赖关系梳理
- asmr_one → local_node 的依赖（获取下载路径、触发扫描、创建播放列表）均为功能性依赖，有 `try/except ImportError` 降级
- local_node → asmr_one 的反向依赖已完全消除
- 下载完成后文件元数据完全自包含：封面、标签、歌词、来源URL 全部写入文件本身
- local_node scanner 从文件提取元数据，不依赖 asmr_one 数据库

### 新增功能

#### 3. 来源URL写入音频标签
- asmr 下载时自动将 `https://asmr.one/work/{work_id}` 写入音频文件元数据
- MP3 → WOAS 帧（Official audio source webpage）
- FLAC → website Vorbis 标签
- MP4/M4A → ©web iTunes 标签
- 来源URL跟随文件走，不依赖数据库

### Bug 修复

#### 4. 封面应用模式修复
- save 模式不再修改音频文件（只保存 cover.jpg），避免 Windows 上 mutagen Bad file descriptor 问题
- embed 模式使用临时文件策略（copy → mutagen save → os.replace），彻底解决 Windows 文件描述符异常
- both 模式同时嵌入封面 + 保存 cover.jpg
- embed 失败时自动回退保存 cover.jpg
- `write_metadata_to_file` 添加 `finally: mf.close()` 防止文件句柄泄漏

#### 5. CoverPickerDialog 默认模式改为 save
- save 模式在 Windows 上最稳定，设为推荐选项

- 开发分支：`dev-260708-decouple-tagwriter-sourceurl`
- 基于 main 分支

## 2026-07-07 元数据行内编辑 + 右侧抽屉批量编辑

### 新增功能

#### 1. TrackTable 行内双击编辑
- 在任意曲目列表中，双击可编辑字段（标题、艺术家、专辑、专辑艺术家、音轨号、年份、流派）的文字区域
- 文字会立即变成输入框，自动 focus + 全选
- Enter 保存、Esc 取消、失焦自动保存
- 可编辑字段悬停时显示蓝色高亮边框提示
- 仅管理员可编辑，非管理员会提示需要管理员权限
- 保存成功后立即更新本地行数据，并 emit `metadata-updated` 通知父组件

#### 2. 右键菜单「编辑元数据」选项
- TrackTable 右键菜单新增「编辑元数据」选项（仅管理员可见）
- 选中多首曲目时右键，会编辑全部选中曲目
- 未选中或只右键一首时，编辑这一首
- 菜单右侧显示当前会编辑的曲目数量（如「3 首」或「单首」）

#### 3. MetadataPanel 右侧抽屉
- 从原来的 Dialog 改造为右侧抽屉式界面
- 显示所有 7 个可编辑字段，按「基础」「专辑」分组
- 多首曲目时：
  - 字段值完全一致 → 显示该值
  - 字段值不一致 → 显示「混合」徽章 + placeholder 提示
- 编辑后字段框变红色作为修改标记：
  - 字段值框边框变红
  - 字段卡片背景变浅红
  - 显示「已修改」红色徽章
  - 出现撤销按钮（可单独撤销该字段）
- 底部显示未保存字段数量，提供「全部撤销」「保存」按钮

#### 4. 未保存修改提示
- 关闭抽屉时若有未保存修改，弹窗询问「保存并关闭 / 不保存关闭 / 取消」
- 切换到其他曲目时若有未保存修改，弹窗询问「保存并切换 / 不保存切换 / 取消」
- 切换场景下会缓存上一次的 tracks，确保正确保存旧数据

### 后端新增

#### API: `POST /api/metadata/batch-update`
- 一次请求批量更新多个字段
- 请求体：`{ track_ids: [int], updates: { field: value } }`
- 所有 track_ids 都会被写入 updates 里的全部字段值
- 返回：`{ updated: int, fields: [str], skipped: [{field, reason}] }`
- 跳过项包含失败原因（如非法整数值、文件回写失败等）

#### 函数: `batch_update_multi_fields`
- 在 `metadata_editor.py` 中新增
- 支持一次写入多个字段，类型转换 + 文件回写 + 事务保护
- 跳过非法值并记录原因，不中断其他字段的处理

### 技术细节

- TrackTable 双击事件 `event.stopPropagation()` 防止冒泡到行级 `onRowDblclick`（避免触发播放）
- MetadataPanel 用 `reactive` 维护字段状态，`computed` 计算 dirty 数量
- 切换场景缓存 `cachedTracks` 确保保存的是旧 tracks 而非新切换的
- `Teleport to="body"` 渲染抽屉避免父容器裁剪
- 抽屉支持 ESC 关闭（含未保存提示拦截）
- 红色标记使用 `ring-red-500/30` + `bg-red-500/5` + `border-red-500/50` 协调配色
