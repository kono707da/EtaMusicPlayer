# 更新日志

## 2026-07-15 — 插件系统改造：移除内置插件，改为在线安装/手动导入

### 变更概要

部署的 Docker 应用不再内置任何插件，所有插件（包括 local_node、asmr_one、bili_audio、shared）都需要在前端「插件管理」页面通过在线注册表安装或手动导入 zip 包。

### Schema 与模型扩展

- **plugins.json schema v2**：新增 `type`（plugin/library）、`dependencies`（数组）、`sha256` 字段
- **shared 库插件**：新增 shared 作为依赖库插件（type=library），asmr_one 和 bili_audio 声明 `dependencies: [{"name": "shared", "version": ">=1.0.0"}]`
- **数据库迁移**：plugins 表新增 4 列：`is_dependency`、`dependent_by`（JSON 数组）、`is_library`、`dependencies`（JSON 数组）

### 动态插件路径发现

- `plugins/registry.py` 改为动态扫描文件系统发现已安装插件，替代硬编码路径
- 新增 `refresh_plugin_paths()` 在启动和安装后扫描 `../../node/` 和 `../../plugins/*/`
- 库类型插件（`is_library=True`）只加入 `sys.path`，不调用 `register()`

### 安装器改造（installer.py）

- **依赖解析**：安装含依赖的插件时自动安装依赖，依赖已安装且版本兼容则跳过，不兼容则更新
- **一次下载多插件**：从 GitHub 下载仓库 zip 包后依次提取依赖和主插件，避免重复下载
- **SHA256 校验**：手动导入时计算并校验完整性
- **路径穿越防护**：拒绝 zip 中的绝对路径、Windows 盘符、`..` 穿越路径
- **元数据校验**：验证插件目录结构（`__init__.py`、`plugin.py`）
- **备份与回滚**：安装前备份旧目录到 `.{name}.bak`，失败时自动恢复

### 手动导入功能

- 后端新增 `POST /api/plugins/import` 路由，支持 multipart/form-data zip 上传（100MB 限制）
- 前端「插件管理」新增「导入」Tab，含拖拽区域和文件选择器
- 自动检测插件名和包名（支持两种目录结构）

### 依赖关系管理

- 删除被其他启用插件依赖的插件会被拒绝，需先删除依赖它的插件
- 删除插件时自动从依赖它的插件的 `dependent_by` 中移除
- 前端表格展示库/依赖 Badge 和依赖关系（被谁依赖、依赖谁）

### Docker 配置改造

- **Dockerfile**：移除 `COPY node/` 和 `COPY plugins/`，改为创建空目录并由 VOLUME 挂载
- **docker-entrypoint.sh**：pip install 只安装 `web/backend/requirements.txt`，插件依赖在插件安装时单独 pip install
- **.dockerignore**：排除 `node/` 和 `plugins/` 目录（运行时安装）
- **Jenkinsfile**：Deploy 阶段 volume 挂载改为 `etamusic-node`、`etamusic-plugins`、`etamusic-data`

### 修改文件

- `plugins.json`：schema 升级到 v2，新增 shared 条目，所有插件加 type/dependencies/sha256
- `web/backend/eta_web/config.py`：`plugins_dirs` 默认值改为空列表
- `web/backend/eta_web/plugins/registry.py`：动态路径发现，新增 add/remove/refresh 函数
- `web/backend/eta_web/plugins/__init__.py`：库类型插件跳过 register()
- `web/backend/eta_web/plugins_manager/models.py`：Plugin 表新增 4 列
- `web/backend/eta_web/plugins_manager/database.py`：自动迁移 4 列
- `web/backend/eta_web/plugins_manager/schemas.py`：PluginOut/OnlinePluginInfo 扩展字段
- `web/backend/eta_web/plugins_manager/online.py`：OnlinePlugin 支持 type/dependencies/sha256
- `web/backend/eta_web/plugins_manager/manager.py`：库类型发现、依赖解析、dependent_by 维护
- `web/backend/eta_web/plugins_manager/installer.py`：依赖解析、SHA256、路径校验、备份回滚、手动导入
- `web/backend/eta_web/plugins_manager/routers.py`：导入路由、删除依赖检查、字段返回
- `web/frontend/src/api/plugin.js`：新增 importPlugin 函数
- `web/frontend/src/views/PluginsView.vue`：导入 Tab、依赖关系展示、库类型 Badge
- `Dockerfile`：移除内置插件 COPY，改为空目录 + VOLUME
- `docker-entrypoint.sh`：移除插件 requirements 安装
- `.dockerignore`：新增，排除 node/ 和 plugins/
- `Jenkinsfile`：volume 挂载更新
- `README.md`：Docker 部署章节更新，新增插件安装说明

## 2026-07-15 — 收集箱职责分离，消除与全部音乐重合

### 变更

- **扫描入库不再自动加入收集箱**：`scanner.py` 扫描新曲目时只加入"全部音乐"系统播放列表，不再重复加入"收集箱"。消除两个系统列表内容完全重合的问题。
- **客户端分组下移除"收集箱"**：`PlaylistTree.vue` 客户端（本机）分组不再显示"收集箱"。收集箱的唯一入口在节点级，语义明确：节点新进的待审曲目临时存放。

### 收集箱职责定义

- **全部音乐**：节点总仓库，扫描和 m3u 导入直接入库
- **收集箱**：节点新进未归类曲目的临时存放，用户检查后操作入库或删除
- **客户端（本机）**：跨节点聚合视图，只保留"全部音乐"+ 用户自建播放列表

### 修改文件

- `node/eta_node/scanner.py`：移除 `_get_or_create_inbox_playlist` 调用和 `_add_to_system_playlist(db, inbox_playlist, track)` 逻辑
- `web/frontend/src/components/PlaylistTree.vue`：客户端分组移除"收集箱"项和相关分支

## 2026-07-14 — 客户端对 node 的版本校验 + 新建播放列表弹窗 + m3u 文件批量导入

### 版本号

- node: `1.0.0` → `1.1.0`
- 客户端: `1.1.0`
- API 协议版本: `1`

### 新增功能

#### 1. 客户端对 node 的版本校验

- **node 端**：新增 `GET /api/version` 接口（无需认证），返回 node 版本号、API 协议版本、要求的最小客户端版本、支持的功能清单
- **客户端**：在连接 node 前进行三重校验
  - **完全不兼容**（API 协议主版本不匹配、node 版本过低、客户端版本低于 node 要求）：**拒绝连接**，在节点管理页显示「版本不兼容」状态并提示原因
  - **核心功能缺失**（auth/tracks/playlists 不可用）：同样**拒绝连接**
  - **部分功能不兼容**（非核心功能如 m3u 导入/去重/音质等缺失）：**允许连接**，但禁用对应 UI 并提示
- **功能能力管理**：客户端维护功能注册表（`config/version.js`），将每个功能映射到 UI 模块。节点管理页提供「详情」按钮查看版本信息与不可用功能列表
- **应用场景**：新增播放列表弹窗中的「从本机 m3u 文件导入」选项，当节点不支持 `import_m3u` 功能时自动禁用并标注「节点版本不支持」

  - [node/eta_node/routers/system.py](node/eta_node/routers/system.py)（新增，`/api/version` 端点）

## 2026-07-14 — 新建播放列表弹窗 + m3u 文件批量导入

### 概述

本次更新重构了新建播放列表的交互流程，新增 m3u/m3u8 播放列表文件导入能力，并优化了侧边栏管理功能的显示策略。

### 新增功能

#### 1. 新建播放列表改为弹窗交互
- **变更前**：点击曲库树分组旁的「+」按钮会直接创建名为「新建播放列表」的默认列表，需要双击重命名。
- **变更后**：点击「+」按钮弹出新建播放列表对话框，用户先输入名称再创建；本机分组仅支持空白播放列表，节点分组额外支持「从本机 m3u 文件导入」选项。
- 涉及文件：
  - [web/frontend/src/components/NewPlaylistDialog.vue](web/frontend/src/components/NewPlaylistDialog.vue)（新增）
  - [web/frontend/src/components/PlaylistTree.vue](web/frontend/src/components/PlaylistTree.vue)（+ 号按钮改为打开弹窗）

#### 2. 节点播放列表支持从 m3u 文件批量导入
- **功能说明**：在新建播放列表弹窗中选择「从本机 m3u 文件导入」模式后，可浏览节点所在主机的文件系统，选择 m3u/m3u8 文件或文件夹进行导入。
- **导入模式**：
  - **单个 m3u 文件**：将该 m3u 引用的所有音频文件按 m3u 顺序导入到指定监控目录，并创建一个播放列表（名为 m3u 文件名或自定义名称）。
  - **文件夹递归**：递归查找所选文件夹下所有 m3u/m3u8 文件，每个 m3u 文件创建一个独立播放列表（名为 m3u 文件名）。
- **复制/剪切**：可选择「复制」（保留原文件）或「剪切」（移动原文件）将音频文件入库到目标监控目录。
- **执行流程**：通过节点任务队列异步执行，依次完成：① 解析 m3u 引用并复制/剪切音频文件 → ② 扫描入库 → ③ 按原顺序创建播放列表与曲目项。任务进度可通过任务队列查看。
- **支持编码**：自动识别 UTF-8 BOM、UTF-8、GBK 编码的 m3u 文件。
- **冲突处理**：目标目录存在同名文件时自动追加 `_1`、`_2` 等后缀避免覆盖。
- 涉及文件：
  - [node/eta_node/m3u_importer.py](node/eta_node/m3u_importer.py)（新增，导入任务处理器）
  - [node/eta_node/task_handlers.py](node/eta_node/task_handlers.py)（注册 `import_m3u` 处理器）
  - [node/eta_node/routers/tasks.py](node/eta_node/routers/tasks.py)（`_ALLOWED_TASK_TYPES` 增加 `import_m3u`）
  - [node/eta_node/routers/watch_dirs.py](node/eta_node/routers/watch_dirs.py)（`browse_directory` 增加 `include_files` 参数，附带返回 m3u/m3u8 文件）
  - [web/frontend/src/api/node.js](web/frontend/src/api/node.js)（新增 `submitM3uImport`，`browseDirectories` 增加 `includeFiles` 参数）

### 优化

#### 3. 隐藏非节点页的管理功能栏目
- **变更前**：侧边栏「管理功能」栏目（用户/权限/去重/音质/审计/任务等）在所有页面均显示。
- **变更后**：仅在「节点管理」页或「管理功能」子页面显示，其他页面（曲库/扫描/元数据等）隐藏，保持工作台简洁。
- 涉及文件：[web/frontend/src/App.vue](web/frontend/src/App.vue)
