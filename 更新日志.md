  - HTTP 状态码 + statusText
  - 后端响应体（JSON 序列化）
  - 请求方法 + API URL（baseURL + url）
  - 错误消息 + 错误类型

- UI 弹窗仍显示简短 description，后端日志记录完整详情

#### 2. 全项目 toast.error 调用点传入原始错误对象
- 所有 `catch (e) { toast.error(...) }` 调用增加第三参数 `e`
- 覆盖文件：App.vue、NodeForm、CoverPickerDialog、MetadataPanel、PathPickerDialog、RenameDialog、TrackTable
- 覆盖 Views：Users、Tasks、Playlists、Settings、Nodes、Dashboard、Scan、Metadata、Permissions、AuditLog、Dedup、Quality、Plugins

## 2026-07-10 修复 chunk 加载失败 + 远程节点 401

### 问题

1. **动态导入失败**：部署新版本后浏览器缓存旧 `index.html`，引用的旧 chunk hash 不存在，点击导航时报 `Failed to fetch dynamically imported module`。
2. **获取播放列表失败 401**：禁用插件后后端重启，远程节点 token 失效，client.js 的 401 自动重登逻辑因远程节点 password 为空无法执行。

### 修复项

#### 1. Router chunk 加载失败自动刷新 (`web/frontend/src/router/index.js`)
- 新增 `lazy()` 函数包裹所有路由的动态 import
- chunk 加载失败时自动 `window.location.reload()` 一次，加载新 `index.html` 获取正确 hash
- 用 `sessionStorage` 标记防止无限刷新循环

#### 2. 应用启动时自动刷新远程节点 token (`web/frontend/src/App.vue`)
- 新增 `onMounted` 钩子：加载插件 → 同步本地节点 → 刷新远程节点 token
- `refreshRemoteTokens()`：遍历所有远程节点调用 `loginRemoteNode` 获取新 token
- 用 `baseUrl` 匹配更新 nodesStore 中的节点（避免 id 格式不一致问题）
- 401/403（认证失败）→ 清除旧 token；其他错误（网络问题）→ 保留旧 token
- 刷新完成后触发 `authVersion++` 让 library store 重新加载

### 改动文件清单
- `web/frontend/src/router/index.js` — lazy import 错误处理
- `web/frontend/src/App.vue` — 启动时自动刷新远程节点 token

## 2026-07-10 节点管理自动健康检测

### 需求

进入「节点管理」页面时自动对每个远程节点执行在线健康检测和登录测试，无需手动逐个点击"测试"按钮。

### 改动项 (`web/frontend/src/views/NodesView.vue`)

#### 1. 自动健康检测
- `loadRemoteNodes` 加载列表后自动调用 `autoCheckAllHealth()`，并行检测所有远程节点
- 每个节点独立处理，互不阻塞（`Promise.allSettled`），单个节点超时/失败不影响其他节点

#### 2. 健康状态管理
- 新增 `nodeHealth` reactive 对象，按节点 id 存储 `{ status, message, checking }`
- 状态：`checking`（检测中）、`online`（在线）、`offline`（离线）、`auth_failed`（认证失败）
- 状态判定逻辑：
  - `success=true` → 在线
  - `success=false` 且 message 含"认证失败/HTTP 4xx/401/403" → 认证失败（节点在线但凭证错）
  - `success=false` 其他 → 离线（连接失败）

#### 3. 测试通过自动登录
- 健康检测通过（online）且节点未登录时，自动静默调用 `loginRemoteNodeAction`（silent 模式，不弹 toast）
- 登录失败不阻塞健康状态展示

#### 4. 表格新增"连接"列
- 检测中：转圈 + "检测中"
- 在线：绿色 ● 在线
- 认证失败：琥珀色 ● 认证失败
- 离线：红色 ● 离线
- 悬停显示具体原因（title 属性，含连接错误/认证详情）

#### 5. 工具栏
- 标题栏新增"重新检测"按钮，可手动触发全量健康检测
- 检测中按钮禁用并显示转圈

#### 6. 手动测试复用健康逻辑
- "测试"按钮改为调用 `checkOneHealth`，测试结果同步写入"连接"列
- 成功时 toast 提示，失败时 toast 显示具体原因

### 改动文件清单
- `web/frontend/src/views/NodesView.vue` — 自动健康检测 + 连接列 + 重新检测按钮

### 分支
- 开发分支：`dev-260710-auto-node-health-check`
- 基于 main 分支

#### 2. standalone 启动方式适配 PyInstaller (`node/eta_node/standalone.py`)

- 此迁移直接修复「加载远程节点失败 500」根因

## 2026-07-09 远程节点下载支持 + 缓存池架构

### 核心问题
用户在访问端配置了代理（如 asmr.one 代理），但远程节点本机没有代理。下载任务在远程节点上执行时，无法通过代理访问外网资源，导致下载失败。

### 解决方案
下载流程重构为「访问端下载 → 本地缓存 → 推送到远程节点」架构：
1. 访问端利用自身代理下载资源到本地缓存池
2. 对缓存文件应用元数据（标签、封面、歌词）
3. 通过 HTTP 上传 API 将文件推送到远程节点
4. 触发远程节点扫描入库
5. 清理本地缓存

### 后端新增

#### 1. eta_node 上传 API (`node/eta_node/routers/upload.py`)
- `POST /api/upload` — 多部分文件上传，支持指定监控目录和子目录路径
- `POST /api/upload/check` — 检查文件是否已存在（支持断点续传/跳过）
- 原子写入：先写 `.part` 临时文件，完成后 `os.replace()` 重命名

- TrackTable 右键菜单新增「编辑元数据」选项（仅管理员可见）

- 未选中或只右键一首时，编辑这一首
- 菜单右侧显示当前会编辑的曲目数量（如「3 首」或「单首」）

#### 3. MetadataPanel 右侧抽屉
- 从原来的 Dialog 改造为右侧抽屉式界面

- 多首曲目时：
  - 字段值完全一致 → 显示该值
  - 字段值不一致 → 显示「混合」徽章 + placeholder 提示
- 编辑后字段框变红色作为修改标记：
  - 字段值框边框变红
  - 字段卡片背景变浅红
  - 显示「已修改」红色徽章
  - 出现撤销按钮（可单独撤销该字段）
- 底部显示未保存字段数量，提供「全部撤销」「保存」按钮

- 关闭抽屉时若有未保存修改，弹窗询问「保存并关闭 / 不保存关闭 / 取消」

- 切换场景下会缓存上一次的 tracks，确保正确保存旧数据

### 后端新增

#### API: `POST /api/metadata/batch-update`
- 一次请求批量更新多个字段
- 请求体：`{ track_ids: [int], updates: { field: value } }`
- 所有 track_ids 都会被写入 updates 里的全部字段值
- 返回：`{ updated: int, fields: [str], skipped: [{field, reason}] }`
- 跳过项包含失败原因（如非法整数值、文件回写失败等）

#### 函数: `batch_update_multi_fields`
- 在 `metadata_editor.py` 中新增
- 支持一次写入多个字段，类型转换 + 文件回写 + 事务保护
- 跳过非法值并记录原因，不中断其他字段的处理

### 技术细节

- TrackTable 双击事件 `event.stopPropagation()` 防止冒泡到行级 `onRowDblclick`（避免触发播放）

- 切换场景缓存 `cachedTracks` 确保保存的是旧 tracks 而非新切换的

- 抽屉支持 ESC 关闭（含未保存提示拦截）
